# Planetarium Project Vision

## Project Direction

This project is moving beyond a simple astronomy demo.

The long-term goal is a healing-focused panoramic experience platform where users can slowly look around a scene, settle into a mood, and stay with it for a while.

The current first theme is the night sky.

## Core Experience

The product should feel like a quiet digital retreat rather than a study tool.

Key principles:

- calm before complexity
- visual immersion over dense data
- interaction only where it improves the mood
- gentle control, not mechanical control
- each theme should feel contemplative, not busy

## First Theme: Night Sky

The night-sky experience is the reference theme for the whole product.

It should support two kinds of use:

1. Passive viewing:
   slow camera movement, quiet atmosphere, fullscreen appreciation
2. Light exploration:
   constellation focus, tracking, observer-based sky changes, sketching

The star theme should be polished enough that future themes can reuse its structure:

- viewer layout
- fullscreen mode
- theme overlay controls
- favorites
- saved states
- ambient sound flow
- slow cinematic camera motion

## Product Identity

Working subtitle:

`밤하늘에 그려봐`

Possible broader identity:

- a healing panorama collection
- a quiet scene viewer
- a theme-based rest experience

The exact brand name can evolve later, but the emotional promise should stay consistent:

`Open a scene, breathe, stay for a while.`

## Design Rules

The product should avoid feeling like:

- a dashboard
- a lesson tool
- a technical observatory console
- a noisy media gallery

Instead, it should feel like:

- an immersive scene
- a slow, elegant screensaver you can guide
- a place to rest your eyes for a few minutes

## Expansion Strategy

The best structure is one site with multiple healing themes inside it, not separate websites per theme.

Why:

- the emotional use case is shared
- the control system is shared
- the brand is shared
- deployment and maintenance stay simpler

That means the current night-sky theme is the foundation, not a side feature.

## Immediate Priority

Before adding more themes, finish the star theme well enough that it can act as the product standard.

Current priority order:

1. finish the night-sky experience
2. stabilize navigation, tracking, and viewing modes
3. improve sketching and favorites
4. define theme architecture
5. expand into new healing panoramas

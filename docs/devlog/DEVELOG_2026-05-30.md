# [Zeravue Build Log #002] 몰입형 감상 경험과 배포 환경 분리 정리

> Building a Quiet Digital Experience

## 날짜

2026-05-30

---

## 오늘의 목표

* Night Sky 화면을 정보 중심 도구보다 조용히 머무는 감상 화면에 가깝게 다듬는다.
* dev/prod 배포 경로가 섞이지 않도록 운영 구조를 명확하게 분리한다.
* 별자리 표기와 검색 경험을 한국어 사용자에게 더 자연스럽게 보정한다.

---

## 진행 내용

### 1. Watch 화면 몰입 흐름 개선

**구현 내용**

* 우측 `현재 시야의 별자리` 카드에서 긴 별자리명이 줄바꿈되며 어색하게 떨어지는 문제를 정리했다.
* 별자리 active-state 문구와 하단 분위기 카드의 설명을 감상 흐름 중심으로 보강했다.
* 몰입 모드를 추가해 좌우 패널을 접고 캔버스 중심으로 감상할 수 있게 했다.

**발생한 문제**

* 별자리 목록과 패널 UI가 정보량을 많이 드러내면서 밤하늘 감상보다 분석 도구처럼 느껴지는 구간이 있었다.

**해결 방법**

* 패널 텍스트 줄바꿈 규칙과 버튼/카드 밀도를 정리하고, 감상 모드에서 보조 UI를 줄이는 방향으로 조정했다.

---

### 2. 별자리 한국어 표기와 검색 보강

**구현 내용**

* `Hercules`를 `헤라클레스자리`로 교정했다.
* `Centaurus`, `Cepheus` 등 통용 표기를 점검해 한글 라벨을 보정했다.
* `헤르쿨레스`, `허큘리스`, `북두칠성` 같은 별칭 검색을 지원하도록 검색 키를 확장했다.

**발생한 문제**

* 영문 원어 기반 표기가 한국어 사용자에게 익숙한 명칭과 다르게 보이는 항목이 있었다.

**해결 방법**

* 표시명은 자연스러운 한국어 표기로 정리하고, 검색은 기존/대체 명칭을 함께 받아들이도록 별칭 매핑을 추가했다.

---

### 3. dev/prod 배포 격리 확인

**구현 내용**

* `dev`는 `/home/ubuntu/Zeravue`, `prod`는 `/home/ubuntu/Zeravue-prod` 경로로 분리했다.
* 각 경로에 `.zeravue-deploy-env`를 두어 잘못된 환경 배포를 차단하도록 했다.
* Docker compose 포트와 nginx 프록시를 확인해 `zeravue.xyz`와 `dev.zeravue.xyz`가 서로 다른 컨테이너로 연결되는지 검증했다.

**발생한 문제**

* 이전에는 main 배포가 dev 경로를 바라보며 우연히 적용되는 구조가 있었고, 운영/개발 경계가 흐려졌다.

**해결 방법**

* 배포 스크립트의 환경 가드와 GitHub Actions secret 검증을 통해 잘못된 경로 조합은 배포 전에 실패하도록 했다.

## 오늘의 회고

### 잘된 점

* 감상 서비스라는 방향성이 더 분명해졌고, UI의 밀도와 문구를 그 방향에 맞춰 줄일 수 있었다.

### 아쉬운 점

* 배포 secret 이름과 경로 분리 기준을 정리하는 과정에서 판단이 흔들려 확인 시간이 길어졌다.

### 배운 점

* 같은 서버에서 dev/prod를 운영할 때 핵심 분리는 host가 아니라 path, sentinel 파일, compose 포트, nginx proxy 조합이다.

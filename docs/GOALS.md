# Zeravue Goals

`Open a scene, breathe, stay for a while.`

## Product Goal

Build Zeravue as a multi-theme healing panorama platform, starting with a polished Night Sky theme as the reference standard.

## Current Focus (Now)

Theme 01: Night Sky quality and stability.

### G1. Viewer Comfort
- Keep camera motion calm, readable, and non-dizzy.
- Make mode differences obvious (space vs observer vs projection).

### G2. Visual Clarity
- Preserve star visibility first.
- Keep constellation guides subtle and supportive.
- Reduce UI clutter on top of the scene.

### G3. Interaction Flow
- Make focus/search/clear actions easy in one place.
- Keep inspector hierarchy simple and scannable.
- Keep sketch interactions lightweight and predictable.

### G4. Release Reliability
- Keep `dev -> merge -> main` workflow strict.
- Keep CI/CD green before merge.
- Keep domain + SSL + renewal stable.

## Definition Of Done (Night Sky Theme)

Night Sky is considered "theme-complete" when:

1. First-time users can understand controls in under one minute.
2. Watching mode feels calm for 5+ minutes without interaction fatigue.
3. Constellation focus and tracking are reliable across view modes.
4. Sketch save/load/favorite flow is stable.
5. UI remains readable on desktop and tablet breakpoints.
6. Deploy pipeline is repeatable with no manual hotfix steps.

## After Theme-Complete

When all criteria above are met:

1. Freeze Night Sky as Theme 01 reference.
2. Start Theme 02 (Aurora Night) using shared systems.
3. Expand to additional healing themes without fragmenting UX.
